一个自动合并代码的简单脚本
链接：https://segmentfault.com/a/1190000039128141

在开发过程中，经常需要将其他分支（如master）的代码，合并到当前开发的功能分支中。

而一个需求的开发，通常会涉及不同的代码仓库。在这些仓库上，会创建同名的功能分支。

如果涉及的代码仓库较多，或频繁合并代码，操作就显得很繁琐。为了更高效的摸鱼，于是写了一个python脚本，用于自动合并代码。

假设D:\rui\work\路径下，存在a-repo、b-repo两个本地代码仓库。在a-repo、b-repo的feature分支上，正在进行需求开发。这时，需要分别将a-repo、b-repo的master分支，合并到feature分支。
